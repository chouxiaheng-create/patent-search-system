设计节 2 / 5 — 数据库设计（Supabase PostgreSQL）

7 张核心表 · RLS 行级权限隔离 · Realtime 订阅驱动动态看板

表结构与关系

① profiles（用户档案）
id uuid FK → auth.users
role enum('admin','user')
display_name text
created_at timestamptz
Supabase Auth 触发器自动创建
② ai_models（AI 模型库）
id uuid PK
owner_id uuid nullable → auth.users
name text（如"Kimi K2.5"）
api_base_url text
api_key_encrypted text
model_id text
is_builtin boolean
usage_type enum('parse','search','report')
owner_id=null 为系统内置；用户可添加私有模型
③ search_strategies（检索策略）
id uuid PK
owner_id uuid nullable → auth.users
name text（如"追踪检索"）
prompt_template text（含 {{占位符}}）
is_builtin boolean
created_at timestamptz
内置3种 + 用户自定义；提示词模板支持修改
④ patent_documents（专利文献）
id uuid PK
user_id uuid → auth.users
file_url text（Supabase Storage）
file_type enum('pdf','docx','xlsx','txt')
parse_status enum(pending/parsing/done/needs_review/failed)
parsed_data jsonb（6项字段+自定义扩展）
quality_warning boolean（复杂排版预警）
user_notes text（手动补充/澄清）
⑤ search_jobs（检索任务主表）
id uuid PK
user_id uuid → auth.users
document_id uuid → patent_documents
status enum(queued/running/completed/failed)
queue_position integer（排队位置）
scheduled_at timestamptz nullable（定时预约）
config jsonb（平台列表/策略列表/结果数量配置）
started_at / completed_at timestamptz
Realtime 订阅此表驱动进度看板
⑥ search_tasks（子任务明细）
id uuid PK
job_id uuid → search_jobs
model_id uuid → ai_models（检索平台）
strategy_id uuid → search_strategies
status enum(pending/running/retrying/done/abandoned)
retry_count integer（最多1次重试）
results jsonb（Top-N 文献列表）
error_msg text nullable
每个节点对应动态看板中的一个流程节点
⑦ reports（检索报告）
id uuid PK
job_id uuid → search_jobs
user_id uuid → auth.users
html_content text（完整 HTML 报告）
selected_docs jsonb（Top-N 文献+元数据）
doc_count integer（实际输出数量）
path_summary jsonb（各路径执行情况摘要）
created_at timestamptz
🔐 RLS 行级安全策略
User 角色
SELECT/INSERT/UPDATE/DELETE
→ 仅限 user_id = auth.uid() 的行
Admin 角色
SELECT/UPDATE/DELETE
→ 所有用户的数据（通过 profiles.role 判断）
