设计节 1 / 5 — 模块总览与技术栈

方案 A：Next.js + Supabase + pg-boss Worker + Supabase Realtime

技术栈选型

前端
  • Next.js 14（App Router）
  • Tailwind CSS + shadcn/ui 组件库
  • React Flow — 动态工作流可视化看板
  • Supabase JS SDK — Auth + Realtime 订阅
  • 部署：Vercel
后端 / 数据层
  • Supabase — Auth / PostgreSQL / Storage / Realtime
  • pg-boss — PostgreSQL 原生任务队列
  • Node.js Worker 服务 — 长任务执行器
  • 部署：Railway（Worker 服务）
  • Next.js API Routes — 轻量 REST 接口
文件处理
  • pdf-parse — PDF 文本提取
  • mammoth — Word (.docx) 解析
  • xlsx — Excel 解析 + 导出
  • docx — Word 格式报告导出
AI 接入层
  • 统一 AIAdapter 接口封装各平台 API
  • 支持 OpenAI 兼容格式 + 自定义格式
  • API Key 加密存储于 Supabase
  • 用户可在界面配置自定义模型接入

六大功能模块

① 文件解析模块
  • PDF / Word / Excel / TXT 输入
  • 调用用户选定大模型提取：
    技术主题、申请人/发明人、申请日
    主要技术方案步骤、核心发明构思
    +用户自定义字段
  • 复杂排版质量预警 + 手动编辑入口
② 检索策略配置模块
  • 3 种内置策略（可编辑提示词）
  • 自定义策略（名称+内容+提示词）
  • 检索平台多选（含自定义接入）
  • 备选文献数量配置（默认5）
③ 并行检索编排模块
  • pg-boss 分发 M×N 个子任务
  • Worker 并发执行，各路径独立
  • 10分钟超时→重试→放弃+记录
  • Realtime 实时推送各节点状态
④ 动态工作流看板
  • React Flow 渲染实时任务节点
  • 每个节点=一条检索路径（平台+策略）
  • 颜色编码：等待/进行/完成/失败
  • Supabase Realtime 驱动状态更新
⑤ 报告生成模块
  • 汇总各路径 Top-N 备选文献
  • 去重 + AI 对比筛选 Top-10（可配置）
  • 注明来源平台/策略/公开时间/链接
  • 输出 HTML，导出 Word/MD/Excel
⑥ 用户与权限模块
  • Supabase Auth（邮箱注册/登录）
  • 角色：Admin / User
  • Admin：可查看/管理所有用户数据
  • User：仅操作自己的数据
  • RLS（行级安全策略）强制隔离
