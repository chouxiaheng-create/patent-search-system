数据库设计修订（基于新约束）

两处变更：① ai_models 表增加能力标签字段 · ② reports.selected_docs 补充来源溯源字段

变更 1 — ai_models 表：新增 capabilities 字段

② ai_models（修订版）
id, name, api_base_url ... （同前）
capabilities jsonb
{
  "deep_reasoning": true/false,
  "web_search": true/false
}
usage_type enum('parse','search','report')
is_builtin boolean
⚙️ 能力要求规则（前端校验 + 后端强制）
解析用模型（parse）
必须：deep_reasoning: true
联网不作要求
检索用模型（search）
必须：deep_reasoning: true + web_search: true
两项缺一不可
报告汇总用模型（report）
必须：deep_reasoning: true
联网不作要求
用户自定义接入模型时，UI 提供能力勾选框；
选择模型时，不符合要求的自动灰显 + 提示原因。

变更 2 — reports.selected_docs：每篇文献增加来源溯源字段

selected_docs: jsonb 数组，每个元素结构：
{
  "rank": 1,
  "title": "文献标题",
  "authors": "作者列表",
  "url": "出处链接",
  "pub_date": "公开时间",
  "relevance_desc": "相关特征描述",
  "citation_gb": "中国国标引文格式",
  "source_platform": "秘塔AI",  // 检索到此文献的平台名称
  "source_strategy": "发明构思检索",  // 所用检索策略名称
  "source_task_id": "uuid"  // 关联 search_tasks 记录 }
报告输出示例：【来源：秘塔AI · 发明构思检索】 标注在每篇文献标题下方
