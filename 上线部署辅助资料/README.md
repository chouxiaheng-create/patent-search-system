## 三、项目核心信息速览（供信息化部快速了解）

| 项目       | 内容                                                            |
| -------- | ------------------------------------------------------------- |
| 项目名称     | 专利检索智能体（Patent Search Agent）                                  |
| 目标用户     | 专利审查员（User 角色）、系统管理员（Admin 角色）                                |
| 核心功能     | 上传专利 → AI 解析 → 多模型 × 多策略并行检索 → 实时看板 → 结构化报告                   |
| 技术架构     | Next.js 16 前端（:3000）+ 独立 Worker（:3001）+ Supabase / PostgreSQL |
| AI 模型接入  | Kimi、智谱、通义千问、DeepSeek、秘塔、OpenAI 兼容接口                          |
| 认证方式     | Supabase Auth（邮箱 + 密码）                                        |
| 报告导出     | Word / Excel / Markdown 三格式                                   |
| 部署形态     | B/S 架构，单位内部署（前端 + Worker + 数据库全部内网）                           |
| 网络暴露     | 仅 443 端口（HTTPS），数据库与 Worker 端口不对外开放                           |
| 是否访问外网   | 是（需调用外部 AI API），由信息化部放行至厂商网关                                  |
| 月均基础设施成本 | 设计目标 ≈ \$0（Render/Vercel/Supabase 免费层；单位内部署后成本仅含服务器与带宽）       |
