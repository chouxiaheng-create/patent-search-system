# 内外网访问情况

> 用于信息化部备案 / 网络拓扑说明

---

## 1. 访问方式总览

| 访问路径     | 适用对象      | 鉴权方式           | 备注               |
| -------- | --------- | -------------- | ---------------- |
| **单位内网** | 在岗审查员     | 内网 IP + 邮箱账号登录 | 主访问路径，优先推荐       |
| **单位外网** | 出差 / 居家办公 | 公网域名 + 邮箱账号登录  | 需经 VPN 或单位反向代理准入 |
| **管理后台** | 系统管理员     | 仅内网 + Admin 角色 | 禁止外网访问           |

## 2. 部署形态

系统采用 **前后端分离 + 浏览器 / B/S 架构**，对外仅暴露 HTTPS（443）端口，无公网 IP 直连数据库。

- **前端服务**：单位内网部署，对内提供 HTTPS 域名 `https://patent.internal.xxx.gov.cn`（示例）。
- **Worker 后端**：内网部署，仅监听内网地址，不对外开放。
- **数据库**：Supabase（自托管 PostgreSQL）或单位内部达梦/人大金仓，部署于内网数据库服务器，与公网完全隔离。

## 3. 网络安全策略

- 出口防火墙仅放行 443 端口的 HTTPS 流量，禁止数据库端口、Worker 端口对外开放。
- 所有 API 调用强制走 TLS 1.2+，禁用明文 HTTP。
- 启用 Supabase RLS（行级安全）策略，用户仅能访问本人数据，管理员除外。
- 外网访问建议通过单位 **VPN + 双因素认证** 准入；如需对部分出差人员开放在线访问，需信息化部审批后开通反向代理白名单。

## 4. 是否接入互联网

**是**。本系统需访问外部 AI 模型 API（Kimi、智谱、通义千问、DeepSeek、秘塔、OpenAI 兼容接口），由信息化部在出口防火墙放行至上述厂商的 API 网关域名（详见环境变量清单）。如需对外访问做严格限制，可改为仅允许特定 IP 出口访问。

> **简明结论**：本系统主要在**单位内网**使用（推荐），经审批后可通过 VPN 或反向代理在**外网**使用，所有流量强制 HTTPS 加密，数据库与 Worker 不对外暴露。
