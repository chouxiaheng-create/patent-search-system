# 账号要求与申请方式

> 用于信息化部备案 / 账号开通流程说明

---

## 1. 是否需要账号登录？

**是**。本系统基于 Supabase Auth 实现邮箱 + 密码认证，**所有功能均要求登录后使用**，未登录用户仅可访问登录页与帮助文档。

## 2. 账号体系

| 角色             | 权限范围                         | 适用范围        |
| -------------- | ---------------------------- | ----------- |
| **User（普通用户）** | 上传专利、发起检索、查阅/导出/删除本人报告       | 专利审查员 / 使用人 |
| **Admin（管理员）** | 查看所有用户数据、用户管理、模型与策略管理、检索任务运维 | 系统管理员       |

## 3. 账号申请流程

### 3.1 普通用户（审查员）

1. **提交申请**：使用人填写《专利检索智能体使用申请表》（附件），提交至部门信息化联络员。
2. **部门审批**：部门负责人签字确认。
3. **管理员开通**：系统管理员在管理后台录入邮箱、初始密码、分配角色（User）。
4. **首次登录**：使用人收到开通邮件，凭初始密码登录后**必须立即修改密码**（系统强制校验）。

### 3.2 管理员

仅由信息化部指派，每个单位建议配备 1–2 名系统管理员，开通流程须经信息化部负责人审批。

## 4. 安全策略

- **密码策略**：最小 8 位、含字母+数字，登录失败 5 次锁定 15 分钟。
- **会话超时**：默认 8 小时无操作自动登出。
- **双因素认证（可选）**：管理员账号强制启用 2FA，普通用户按需开启。
- **审计日志**：登录、检索任务创建、报告导出、模型/策略变更均记录操作日志，保留 ≥ 180 天。
- **离职清理**：人员离职 3 个工作日内由管理员禁用账号并归档数据。

## 5. 联系方式

- **系统管理员**：信息化部 XXX，电话 XXX-XXXX
- **账号申请邮箱**：xxx@xxx.gov.cn
- **申请表附件**：《专利检索智能体使用申请表》（信息化部备案留存）

> **简明结论**：需要账号登录；普通用户由部门申请、管理员开通；强制密码策略 + 审计日志 + 离职清理。
