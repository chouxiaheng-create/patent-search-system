# /verify Skill

在声明任何计划、功能或修复"完成"之前，必须执行以下验证清单。未通过全部检查前，**禁止声称完成**。

## 验证清单

### 1. 类型检查
- [ ] 运行 `npx tsc --noEmit`（前端根目录和 worker 目录分别执行）
- [ ] 无类型错误、无缺失导入、无默认导出不匹配

### 2. 构建验证
- [ ] 运行前端构建：`npm run build`
- [ ] 运行 Worker 构建：`cd worker && npm run build`
- [ ] 构建命令返回 0，无 ERROR 级别日志

### 3. 产物质量检查
- [ ] 检查 `dist/` 或 `.next/` 中是否出现平台注入物（水印、未知 CDN 脚本、统计代码）
- [ ] 检查构建产物中的资源路径（JS/CSS 引用）是否为相对路径 `./` 而非绝对路径 `/`
- [ ] 若发现注入物，调查构建工具/部署平台行为，而非修改源码试图消除

### 4. 预览验证
- [ ] 使用本地服务器预览：`npx serve dist` 或 `npm start`，**禁止用 file:// 直接打开 HTML**
- [ ] 在浏览器中打开并检查 Console，确认无 404 资源加载错误
- [ ] 测试该功能的核心交互路径（ golden path ）

### 5. 数据库与迁移
- [ ] 若修改了 schema/migrations，确认 SQL 是幂等的（可重复执行）
- [ ] 检查连接池配置，确认不会导致 pg-boss 等工具耗尽连接

### 6. AI 模型适配器验证
- [ ] 若修改了 `worker/src/adapters/`，确认配置来源于官方文档或用户提供的参考资料
- [ ] 若无法调用真实 API，切换至 `MOCK_MODE=true` 验证适配器接口和数据流正确

### 7. 回归检查
- [ ] 确认本次修改没有破坏已有功能（尤其是上次验证通过的功能）

## 失败处理
- 任何一项失败 = 未完成。修复后重新运行完整清单，不要跳过已通过的项（除非确定无副作用）
- 若构建/预览因平台环境差异失败（Windows CMD 路径、绝对路径等），优先修复环境兼容性问题，而非绕过
